01

// INSTALL

Clone the Pharos Agent Dev Suite and install dependencies. Requires Node.js ≥18 and Foundry for contract compilation.

$ git clone https://github.com/AnomalyCo/Pharos.git
$ cd Pharos
$ npm install

Set your private key and RPC endpoint. Keys are never logged or exposed.

$ export PRIVATE_KEY=0x...
$ export RPC_URL=https://rpc.atlantic.pharos.network
⚠ PRIVATE_KEY is read directly by forge and viem — never printed or committed.

// ARCHITECTURE

// DUAL CASCADE

The defining feature of Pharos ADS. Two cascading layers that bridge AI agents to blockchain:

  1. Skill Cascade — 42 subskills in skill/subskills/. Each subskill is a markdown file with context, examples, and conventions. The agent reads these to understand Pharos-specific patterns (pull-over-push, gas stipend rules, etc.).
  2. Tool Cascade — 18 MCP tools in mcp-server/index.js. Each tool wraps a viem RPC call with input validation, error handling, and timeouts. The agent invokes these to execute on-chain.

// STACK

Node.js + ESM RUNTIME
viem v2 RPC
MCP SDK v0.6 PROTOCOL
Foundry COMPILER
stdio transport IPC

// LIMITS

  • No 2300 gas stipend on native PHRS transfers — use pull-over-push pattern
  • RPC rate-limited: ~30 req/s sustained, ~100 req/s burst
  • eth_getLogs limited to 100 block range per request
  • safe and finalized block tags available for production reads
  • eth_getAccount is Pharos-specific (returns balance, nonce, codeHash, storageRoot)
  • debug_traceTransaction publicly enabled on both networks
  • Blockscout verifier mode — no API key needed on testnet
Chain ID 688689 (NOT 688688). This is a common pitfall — double-check before deploying.
