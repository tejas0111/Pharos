# Localization and Copy

Adjust product copy, labels, and localization structure for clearer user-facing text.

## When to Use

localization, i18n, translation, copy, microcopy, labels, strings, text content, locale, multi-language

## When NOT to Use

UI component changes (use react-ui-patterns-and-hooks or tailwind-shadcn-ui-workflow), or technical documentation (use docs-and-example-generation)

## Workflow

1. Identify the user-facing strings and target locales or tone.
2. Draft improved copy or localization structure.
3. Show the plan and proceed once it looks right.
4. Keep the wording aligned with the product context and flow.

## Output

- copy draft
- localization notes
- string list
- tone guidance

## Examples

- "Improve the copy for this wallet connect flow"
- "Prepare the localization structure for a future multi-language release"
- "Review the error messages in this contract for clarity and user-friendliness"

## Verification

Visual review of the text in context.
