# Release Notes and Changelog

Use when the user needs a polished release summary.

## Workflow

1. Summarize the user-facing changes and risks.
2. Draft release notes or changelog entries in clear language.
3. Show the outline and proceed once it looks right.
4. Keep the notes tied to the actual diff and not marketing fluff.

## Output

- release note draft
- changelog entry
- risk notes
- summary bullets

## Gate

Low risk. Present the outline first, then proceed once it looks right.
