# Contract Architecture

Use when the user needs a contract-level system design before code is written.

## Workflow

1. Capture the product goal, trust model, and constraints.
2. Propose module boundaries, storage layout, events, and permission model.
3. Present the plan and ask whether it needs changes.
4. After approval, produce architecture notes or a code-ready breakdown.

## Output

- module map
- storage and permission plan
- risks and tradeoffs
- next implementation step

## Gate

High risk. Do not write contract code before approval.
